# does-it-work

Скилл для Claude Code / Codex, который отвечает на главный вопрос вайбкодера:
**«а оно вообще работает?»**

Прогоняет ваше живое приложение автотестами, находит баги (репорт с severity:
что критично, что терпит), отвечает «можно ли в прод» — и оставляет каркас
автотестов как защиту от регрессий при следующем вайбкодинге.

## Что умеет

- **Аудит продукта**: smoke-прогон ядра, карта покрытия, баг-репорт с severity
  и честный вердикт — что работает, что сломано, что не проверялось.
- **Генерация тестовых фреймворков** по пяти проверенным веткам:

| Ветка | Стек |
|---|---|
| API, Python | pytest + httpx + Pydantic + Allure |
| API, Java | JUnit 5 + REST Assured + Maven |
| WEB, Python | Playwright + Page Object |
| WEB, Python (легаси) | Selenium + Page Object |
| WEB, Java | Selenide + JUnit 5 |

- **Конвертация** OpenAPI / Postman / curl / HAR в тесты; негативные кейсы
  и проверка контрактов.
- **Стабилизация флакающих тестов** и **ревью качества** уже написанных тестов.

Оформление: имена функций — латиницей, docstrings / Allure-тайтлы / шаги /
сообщения ассертов — на русском.

## Установка

### Claude Code

Глобально (скилл доступен во всех проектах):

```bash
git clone https://github.com/TsakunovR/does-it-work ~/.claude/skills/does-it-work
```

Или только в текущий проект:

```bash
git clone https://github.com/TsakunovR/does-it-work .claude/skills/does-it-work
```

Claude Code подхватит скилл при следующем запуске. Проверить — команда `/help`
или просто попросить «проверь мой продукт».

```bash

```

Формат `SKILL.md` — открытый стандарт, поэтому один и тот же скилл работает в обоих
инструментах.

## Примеры запросов

Скилл активируется сам — просто опишите задачу своими словами:

- «Я навайбкодил приложение — проверь, что оно реально работает» —
  аудит: smoke-прогон, баг-репорт с severity, вердикт о готовности к проду.
- «Найди баги в моём API: https://.../openapi.json» —
  проверка живого стенда против спеки, расхождения — в отчёт.
- «Напиши API-тесты по этой OpenAPI-спеке» —
  каркас + тесты с нуля (Python по умолчанию, Java — по запросу).
- «Покрой тестами эндпоинты бронирования, вот curl-примеры» —
  тесты из curl / Postman-коллекции / HAR.
- «Сгенерируй UI-тесты на Playwright для страницы логина» —
  web-ветка с Page Object и логином через API.
- «Добавь негативные кейсы и проверку контрактов к существующим тестам» —
  дописывание в ваш проект, без разворачивания каркаса.
- «Тесты флакают в CI, стабилизируй» — режим стабилизации:
  воспроизведение повторами → диагноз → починка причины → доказательство прогонами.
- «Сделай ревью качества наших автотестов» — режим ревью:
  запуск, чек-листы, отчёт с находками по severity.

## Как это работает

Триггеры и полный порядок работы — в [`SKILL.md`](SKILL.md).

Структура:

- `SKILL.md` — точка входа: выбор ветки, порядок работы, конвенции, red flags.
- `templates/` — пять эталонных каркасов, проверенных запуском против живых стендов.
- `reference/` — методички: источники тест-кейсов, паттерны живого API, стабилизация и ревью.
- `scripts/` — префлайт-проверка стенда и bootstrap окружения.
- `examples/` — образцы карты покрытия и баг-репорта.
