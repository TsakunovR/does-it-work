# Пример карты покрытия (docs/coverage.md в сгенерированном проекте)

Стройте такую таблицу на шаге 2 для API с 15+ операциями, согласуйте объём
с пользователем и кладите в `docs/coverage.md` проекта. Статусы:
✅ покрыто · 🔜 запланировано · ⏸ осознанно отложено (с причиной) · 🚫 не тестируем.

## Правила ведения

- Каждая строка спеки должна попасть в таблицу — «не тестируем» тоже решение, с причиной.
- Обновляйте при каждой доработке; расхождение карты и тестов — повод для ревизии.
- Итоговые цифры (N операций: X покрыто, Y отложено) — в отчёт пользователю.
